TOURIST MODE INTERFACE
The tourist interface focuses on safety, travel guidance, and emergency response.
Main Dashboard Components include:
Safety Score Card: Shows real-time safety level based on location and current events.
Live Safety Map: Interactive map showing current user location, safe zones, danger zones, community responders, police stations, and hospitals.
Quick Emergency Controls: Large Intelligent SOS button with activation countdown.
Safety Pulse Status: Shows last check-in confirmation.
Recent Alerts Feed: Displays hazard warnings and safety notifications.
CORE SAFETY & EMERGENCY RESPONSE (THE SAFE LAYER)
Intelligent SOS & SMS Fallback
A one-tap SOS trigger. When activated, it transmits live GPS location, medical profile, and an emergency signal. If the internet fails, an encrypted SMS fallback automatically sends GPS coordinates to official admin numbers. The UI shows an SOS countdown animation, signal transmission indicators, and SMS fallback status.
Offline Sentinel Tracking
When internet connectivity is lost, the interface switches to Offline Sentinel Mode. Features include offline maps, Latitude/Longitude extraction, SMS transmission logs, and admin sync confirmation. The UI clearly indicates the user is in offline safety monitoring mode.
Emergency Safe Zone Navigation
Interactive routing system guiding the tourist to the nearest police stations, hospitals, embassies, and safe shelters. The route planner prioritizes safe environments rather than the fastest routes.
Safety Pulse Check-In & Community Escalation
A scheduled check-in system asks: "Are you safe?". If the user fails to respond, the system automatically triggers a rapid-response broadcast to local authorities, postal workers, government networks, and nearby app community members with the user's last active location. The UI visualizes escalation stages.
Real-Time Hazard Intelligence
A live intelligence feed monitors news, local incidents, civil disturbances, and weather risks. Hazards appear on the safety map with risk scores, acting as a proactive helper to the tourist.
Dynamic Geofencing Check
The system constantly checks the tourist location against restricted zones. If the tourist approaches a high-risk area, the interface displays a warning banner and suggests safer routes.
Adaptive Battery Optimization & Dead-Battery Escalation
An intelligent system dynamically adjusts GPS tracking frequency depending on battery level and risk conditions. If the battery dies, a final distress signal with the last live location is broadcast to the community and authorities for immediate recovery.
AI TRAVEL ASSISTANCE & IMMERSIVE EXPERIENCE
Conversational AI Concierge
A floating AI assistant supporting text chat and voice commands. Capabilities include cultural etiquette guidance, emergency translation, local recommendations, and safety tips.
Safe Route Itinerary Planner
A route planning interface that generates safety-optimized travel paths. Routes prioritize well-lit areas, crowded streets, and verified safe zones. Users can plan daily travel activities.
Avatar Virtual Companion
An immersive feature where friends or family can join the tourist journey. The interface shows camera streaming, live map sharing, and a voice connection, allowing loved ones to virtually accompany the tourist as a companion.
Verified Local Guide Network
A marketplace-style interface listing verified guides certified as local experts, first responders, and cultural navigators. Tourists can request guide assistance directly from the interface.
ENTERPRISE MODE OVERVIEW
Enterprise Mode is designed for travel agencies, tour companies, and safety administrators. The UI must resemble a professional command center dashboard.
Enterprise Onboarding & Linking
Companies register with their details to receive an Enterprise Command Dashboard. Tourists who install GuardianAI are linked via company codes, tour plans, or linked emails. The dashboard displays a list of tourists including Name, Email, Country, Trip Status, Safety Status, Battery Level, Connectivity Status, and Last Known Location, complete with filtering tools.
Trip Planner Dashboard
Operators create travel plans with fields for Trip Name, Destination, Dates, Assigned Tourists, Tour Guide, Emergency Contacts, and Risk Level. This transforms into a live monitoring dashboard.
Group Safety Dashboard
A single command view displaying a live tourist map, status cards, online/offline indicators, and safety alerts. Statistics cards show total tourists, active/offline status, safety alerts, and SOS events.
Individual Tourist View
Clicking any tourist opens a detailed profile displaying live GPS tracking, movement history, battery health, connectivity status, medical profile, emergency contacts, recent alerts, and safety pulse responses. Operators can trigger manual check-ins.
Community & Authority Integration
The dashboard displays nearby GuardianAI community members (trusted volunteers, verified guides, app users) on the map. It also connects with emergency authorities, showing nearby police and hospitals. During an SOS, the system displays the tourist's location, incident details, notified authorities, and response progress.
Command Center Map & Activity Feed
The centerpiece is a large real-time map displaying tourists, safe/danger zones, community responders, and emergency facilities using color-coded markers. Alongside it, a real-time event timeline shows system activity (e.g., tourist offline, SOS activated, missed safety check).
WORKFLOW OF THE APPLICATION
Phase 1 – Pre-Trip Setup
User installs GuardianAI, selects Tourist Mode, enters emergency contacts and medical information, connects friends for the virtual companion, and plans routes using the AI itinerary planner.
Phase 2 – Active Travel
Tourist explores the destination. Background safety systems monitor location, risk zones, hazards, and battery health. The AI concierge provides guidance.
Phase 3 – Offline Mode
If the internet disappears, Offline Sentinel activates. Offline maps continue working, and GPS coordinates are sent via encrypted SMS to admin servers.
Phase 4 – Emergency Scenarios
Unresponsive User: Missed safety pulse triggers community and authority escalation.
Dead Battery: Final distress signal sent to authorities before shutdown.
SOS Activation: Emergency routing begins, SMS fallback initiates if needed, and authorities/admins are notified.
Phase 5 – Enterprise Response
The enterprise command center receives alerts. Operators view tourist location, trip data, and emergency status, and can coordinate with verified guides and authorities for recovery.
